# Page Template Reference

## 1) Root Service-City SEO Page

Pattern:

- `/{service-slug}-{city-slug}`

Template:

- `src/templates/serviceCityTemplate.js`

Blocks:

1. Hero with city + service H1
2. Service explanation from rich service metadata
3. Local relevance section from city metadata
4. Benefits/materials/process sections
5. Project examples
6. FAQ section
7. Internal linking lattice
8. Conversion CTA

Schema:

- `FAQPage`
- `Service`
- plus global `WebPage`, `BreadcrumbList`, `LocalBusiness`, `Organization`

## 2) Root City Hub

Pattern:

- `/{city-slug}-carpentry`

Template:

- `renderSeoCityHub` in `src/templates/seoHubTemplates.js`

Blocks:

1. City-specific intro and notes
2. Full service list for that city
3. Nearby city hub links
4. Project references
5. City-focused FAQ
6. CTA

Schema:

- `FAQPage`
- `Service` (city-level service context)
- global page schema stack

## 3) Primary Service Hub

Pattern:

- `/services/{service-slug}`

Template:

- `renderSeoServiceHub` in `src/templates/seoHubTemplates.js`

Blocks:

1. Service-specific H1 and long description
2. Use cases / benefits / materials / process notes
3. Links to all city pages for that service
4. Related service links
5. Project links
6. FAQ
7. CTA

Schema:

- `FAQPage`
- `Service`

## 4) Legacy Service-City Page

Pattern:

- `/services/{service-slug}-{city-slug}`

Template:

- `src/templates/serviceCityPage.js`

Note:

- Still generated by the existing architecture.
- Coexists with root-level SEO service-city routes.

## 5) Project Page

Pattern:

- `/projects/{project-slug}`

Template:

- `renderProjectPage` in `src/templates/pageBuilders.js`

Blocks:

1. Project overview
2. Image gallery with SEO alt/caption
3. Scope of work
4. Materials
5. Before/after narrative
6. Internal links to matching service/city pages
7. CTA

Schema:

- `ImageObject` per image
- global page schema stack

## 6) Blog Page

Pattern:

- `/blog/{post-slug}`

Template:

- `renderBlogPost` in `src/templates/pageBuilders.js`

Blocks:

1. Intro + practical guidance
2. Supporting section content
3. Internal links to money pages and hubs
4. CTA

## 7) Core Pages

Core routes:

- `/`
- `/services`
- `/projects`
- `/cities`
- `/blog`
- `/about`
- `/contact`

Legacy route behavior:

- `/home` and `/home/` permanently redirect to `/`.

## Breadcrumb Strategy

Custom breadcrumbs are explicitly set on:

- root service-city pages
- root city hubs
- primary service hubs

Fallback breadcrumbs are still auto-generated for other routes.
